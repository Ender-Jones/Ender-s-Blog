# Console 行为契约(/console/)

> 这份文件回答"什么情况下会发生什么、我会看到什么". 它是 console 的设计文档 —
> **以后改 console 行为, 先改这里, 你点头了再动代码**. 与本文件不符的行为都算 bug.
> (2026-07-11 立此存照: 之前没有这份文档, console 对印是黑箱 — 这是流程失误, 不再犯.)

## 一句话模型

你在一块**当日工作区**(D1)上一行一行记 → 睡前一个**仪式**(close the day)把这一天
编成 markdown 提交进 git(唯一本源)→ 站点自动重建. git 里没有的, 都还不算数.

## 录入前缀(打在行首, 决定这行进哪个段)

| 前缀 | 字段 | 会触发什么 | 不会触发什么 |
|---|---|---|---|
| (无) | done | 纯归档 → 当天 `### Done` | — |
| `@` | project | 纯归档 → `### Project`. 语义 = "今天挂在哪条线上"的上下文标签, 给几个月后的自己扫读用 | **不**进 carry-over, 不追踪, 没有开始/完成状态 |
| `=` | decision | 纯归档 → `### Decision` | 同上 |
| `!` | risk | 纯归档 → `### Risk` | 同上 |
| `>` | next | **唯一有行为的字段**: 明天它会出现在 CARRY-OVER 卡里, 逼你给出口 | — |
| `~` | dropped | 归档 → `### Dropped`(删除线, 有尊严的退场) | 录入台没给它 chip — drop 通常发生在分诊, 不在录入 |

## 仪式字段(只在关日时问一次, 不是录入前缀)

- **public**: 一句公开话 → `### Public` → 主页墙下/worklog 页顶/about 三处 NOW 行. 可空.
- **energy**: 1–5 → `<!-- energy: n -->` 注释. 可空 — 找规律用, 不是打分.
- **save point**: 不是新字段 — 它只是帮你补记一条 `> next`. 当天**已有 next 时输入框消失**
  (已备存档点). **非必填**: 空着照样能关日, 代价只是明天没有零思考起点.

## 闸门(只有这两道, save point 不是闸门)

1. **carry-over 分诊闸**: 昨天的 next 没给出口(done / still next / drop)→ 不准关今天. 刻意的.
2. **空日无 close**: 一行都没有就没有"关日"可言.

## 日界与关日

- **04:00 JST 翻日**: 它只决定"这一行算哪天的"(熬夜到凌晨三点算昨天), **没有任何自动关日**.
- **close 永远是手动仪式**, 设计意图 = 今天的最后一个动作, 关完就去睡.
- 忘了关也不炸: 行都在 D1 里躺着 — 但见下面边界 2.

## 已知边界(诚实清单, 2026-07-11 版)

1. **关日后、同一逻辑日内再录入**: 行能进 D1, 但永远到不了 git(再点 close 会被
   409 "already closed" 拒绝), 成为孤儿行. 前端刷新后也不知道"今天已关过".
2. **忘记关日、睡过 04:00**: 昨天的行悬在 D1 工作区; console 只看"今天", 前端没有
   "补关昨天"的入口(后端其实支持指定日期).
3. **关日当晚 carry-over 回显旧项**: carry 候选烤在构建产物里且只取"今天之前"的日子,
   分诊状态随归档消失 → 关日后旧卡复活. 次日自愈. 此时**别点分诊钮**(会触发边界 1).
4. **push 失败无离线缓冲**(已定论): 玫瑰色报错 + 重试钮, 不假装存上了.

## 拟定决议(2026-07-12 商量收敛; 印最终点头后生效并上移正文, Claude 才动代码)

**关日后的世界 — 采印的方向: 关日即翻页(roll-forward)**

原则: **录入台永不禁用**. 捕捉随时可用是这个工具的第一戒律 — "爬起来又干活"被工具拒绝
记录, 才是对 ADHD 最大的背叛. (据此否决早先的"整日禁用"提案.)

1. close 成功那一刻, "今天"提前结束: 之后录入的每一行自动归**下一个逻辑日**(D1 直接记
   明日日期). 睡前爬起来干的活, 明早打开 console 已经躺在工作区里.
2. **一个逻辑日最多关一次**: 关日钮在 04:00 真正翻日前保持锁定, 显示
   "day closed — next close unlocks at 04:00". 这就堵死了"连按几次 close 穿越到几天后".
3. **不做 unclose**: 仪式不可撤销 — 撤销要改已提交的 git 段落, 复杂度不值; 真要改,
   手写 md 是永远在的兜底. (close 前多犹豫一秒即可, 反正没有任何东西逼你在某时刻关日.)
4. UI 契约: 关日后捕捉卡切换到 "EARLY START — logging for <明日日期>" 状态; 列表显示
   明日已预录的行; carry-over 卡隐藏(分诊是明天开日的事); md 预览显示明日的块.
5. 忘关睡过 04:00(边界 2): 开日时检测"昨日有未归档行" → 顶部横幅一键**补关昨天**
   (后端本就支持指定日期). 补关走完整仪式(recap/save point/public/energy 都还能填).
6. 实现落点: `GET /api/log` 返回该日 `closed` 标志 + 服务端 POST 时"今日已关则记明日";
   前端按标志切 EARLY START. 边界 1/3 随之消失.

**project 字段 — 职能保留, 词暂不动**

- 语义印已接受: "今天挂在哪条线上"的上下文标签, 纯归档无行为; done 才是"干成了什么".
- 改名成本: 解析器要兼容新旧两代段名 + 已归档的 `### Project` 永不回改 → 成本>收益.
- 若将来真要换, 候选是 `thread`(与站点 the-line / public_thread 词汇同族), 放到换域名级
  的大扫除里顺手做, 不单独做.
