# Console 行为契约(/console/)

> 这份文件回答"什么情况下会发生什么、我会看到什么". 它是 console 的设计文档 —
> **以后改 console 行为, 先改这里, 你点头了再动代码**. 与本文件不符的行为都算 bug.
> (2026-07-11 立此存照: 之前没有这份文档, console 对印是黑箱 — 这是流程失误, 不再犯.)

## 一句话模型

你在一块**当日工作区**(D1)上一行一行记 → 睡前一个**仪式**(close the day)把这一天
编成 markdown 提交进 git(唯一本源)→ 站点自动重建. git 里没有的, 都还不算数.

## 录入前缀(打在行首, 决定这行进哪个段)

> 每个词的行为注解**常驻录入台 SYNTAX 区**(写的时候看得到) — 本表只做深规格.

| 前缀 | 字段 | 会触发什么 | 不会触发什么 |
|---|---|---|---|
| (无) | done | 纯归档 → 当天 `### Done`. 语义 = **结果**: 摸完了的事 | — |
| `@` | wip | 纯归档 → `### WIP`(旧名 `### Project`, 2026-07-12 全库改词完成). 语义 = **投入**: "今天在摸什么", 开始摸就记; 摸完了另记一条 done — **和 done 内容重复是特性**(早上 wip A, 晚上 done A); 没摸完就自然没有 done, 这条 wip 就是"努力过"的证据. 记录时零判断, 不用预测做不做得完 | **不**进 carry-over, 不追踪 |
| `=` | decision | 纯归档 → `### Decision` | 同上 |
| `!` | risk | 纯归档 → `### Risk` | 同上 |
| `>` | next | **唯一有行为的字段**: 明天它会出现在 CARRY-OVER 卡里, 逼你给出口 | — |
| `~` | dropped | 归档 → `### Dropped`(删除线, 有尊严的退场) | 录入台没给它 chip — drop 通常发生在分诊, 不在录入 |

## 仪式字段(只在关日时问一次, 不是录入前缀)

- **public**: 一句公开话 → `### Public` → 主页墙下/worklog 页顶/about 三处 NOW 行. 可空.
- **energy**: 1–5 → `<!-- energy: n -->` 注释. 可空 — 找规律用, 不是打分.
- **save point**: 不是新字段 — 它只是帮你补记一条 `> next`. 当天**已有 next 时输入框消失**
  (已备存档点). **非必填**: 空着照样能关日, 代价只是明天没有零思考起点.

## 闸门(只有这两道, save point 不是闸门)

1. **carry-over 分诊闸**: 昨天的 next 没给出口(done / still next / drop)→ 不准关**今天**. 刻意的.
   (补关旧日不过此闸 — 那一天的分诊时刻已经过去了, 直接进仪式.)
2. **空日无 close**: 一行都没有就没有"关日"可言.

## 日界与关日(关日即翻页, 2026-07-12 实现)

原则: **录入台永不禁用** — 捕捉随时可用是这个工具的第一戒律, "爬起来又干活"被拒绝记录
才是对 ADHD 最大的背叛. 印对这套模型的原话: "完美的符合我对关日的所有想象".

- **04:00 JST 翻日**: 它只决定"这一行算哪天的"(熬夜到凌晨三点算昨天), **没有任何自动关日**.
- **close 永远是手动仪式**, 设计意图 = 今天的最后一个动作, 关完就去睡.
- **关日即翻页**: close 成功那一刻"今天"提前结束 — 之后录入的每一行自动归**下一个逻辑日**
  (server 判定, 开着的旧标签页也骗不过). 睡前爬起来干的活, 明早打开 console 已经躺在工作区里.
- **一个逻辑日最多关一次**: 关日钮变 "day closed — next close unlocks at 04:00" 并锁定,
  堵死"连按 close 穿越到几天后". 后端双保险: 已关日 409, 未来日不可关.
- **不做 unclose**: 仪式不可撤销; 真要改已提交的段落, 手写 md 是永远在的兜底.
- **UI 把这一切显在脸上**: 关日后日期条翻到明日, 状态旗 "EARLY START — <今日> closed ·
  logging for <明日>", 列表显示明日已预录的行, carry 卡隐藏, md 预览显示明日的块.
- **忘关睡过 04:00**: 开日检测到旧日还有未归档行 → 顶部 amber 横幅一键**补关那一天**
  (完整仪式: recap / save point / public / energy). 漏了多天就从最旧的开始一次补一天.
  补关模式下录入台照常可用, 行回填到那一天(补记漏行是特性); save point 也落在那一天的
  `### Next` 里, 站点重建后自然出现在今天的 CARRY-OVER.
- 04:00 当刻页面还开着: 空闲时(没在打字、仪式没开)自动刷新翻日; 在打字就等输入清空再刷.

## 已知边界(诚实清单, 2026-07-12 版)

1. **push 失败无离线缓冲**(已定论): 玫瑰色报错 + 重试钮, 不假装存上了.
2. **绕过 console 手写 md 收掉的日子**: 该日在 D1 看来从没关过 — 若里面还悬着未归档行,
   横幅会催补关, 而 close 会撞 git 检查 409(该日在月文件里已存在). 出口: 去 D1 控制台
   删该日的行, 或无视横幅.
3. **旧模型时代的孤儿行**(关日后又录入的行, 2026-07-12 前产生): 不出横幅、也永远到不了
   git — 要救就去 D1 控制台捞. 新模型下不再产生这种行.
4. 补关推送后, 今天的 CARRY-OVER 要等站点重建完(几分钟)才能看到那一天的 next.

(2026-07-11 版边界 1/2/3 — 关日后孤儿行、漏关无入口、关日当晚 carry 卡复活 — 已随
关日即翻页的实现消失.)

## 裁决记录(2026-07-12, 均已实现)

- **关日即翻页**: 规格见上文「日界与关日」. 实现落点: `GET /api/log` 返回
  `closed / effectiveDay / staleDays`; POST 在"今日已关"时由 server 落到下一逻辑日;
  "某日已关" = 该日存在 `committed=1` 的行(空日不能关, 推断严密, D1 零迁移).
- **project → wip**: 印指出 project/done 易重复且**迁移成本不进裁决标准, 唯一准则 =
  对抗 ADHD 最优**; Claude 曾提"完不成才写 wip"的互斥规则 → 被印的版本取代(那条规则
  要求记录时预测"今天做不做得完" = 多余决策, 印的版本零判断, 更优). 定案语义见上文
  字段表, 用法注解常驻录入台 SYNTAX 区. 全库迁移完成: 2026-04/05/07 词段(41 处)、
  两代解析器(旧词保留为别名)、console chip、md.js、D1 读取归一; "legacy 原文永不回改"
  在**词段维度**由印本人解除(2026-07-12), 内容行一字未动.
